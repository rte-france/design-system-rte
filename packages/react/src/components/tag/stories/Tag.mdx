<div className="Guidelines">

        <h2 id="guidelines" className="h2" style={{border:"none", fontFamily:"Nunito", color:"#2964A0"}} > Usages </h2>

        <h3 id="usage-examples" className="h3" style={{fontFamily:"Nunito"}}> Usages Guidelines : </h3>

        <p>Les principales règles qui entourent l'utilisation du composant :</p>

        <p className="no-margin-bottom">Utiliser un Tag pour indiquer un état ou une catégorie</p>
        <p className="light-gray no-margin-top">Le Tag doit toujours représenter une <b>information contextuelle statique</b> : état ("En cours", "Fermé"), priorité ("Urgent"), type ("PDF").</p>

        <p className="no-margin-bottom">Ne jamais rendre un Tag interactif</p>
        <p className="light-gray no-margin-top">Si un clic est attendu (ex : pour filtrer ou supprimer), utiliser un Chip à la place. Le Tag est <b>purement informatif.</b></p>

        <p className="no-margin-bottom">Éviter les phrases longues</p>
        <p className="light-gray no-margin-top">Les Tags doivent être courts (1 à 2 mots). Pas de phrases ni de ponctuation.</p>

        <p className="no-margin-bottom">Utiliser une couleur pour hiérarchiser les statuts</p>
        <p className="light-gray no-margin-top">Les couleurs servent à renforcer la hiérarchie visuelle (ex : vert = validé, rouge = erreur). Ne pas utiliser de couleur sans signification métier.</p>

        <p className="no-margin-bottom">Ne pas empiler trop de Tags</p>
        <p className="light-gray no-margin-top">Afficher un maximum de 2 à 3 Tags par élément. En cas de dépassement, utiliser un overflow ("+2 autres").</p>

        <h3 id="content-standards" className="h3" style={{fontFamily:"Nunito"}}> Content Standards : </h3>

        <p>Quels sont les standards de contenu ? (UX Writing)</p>

        <p className="no-margin-bottom">Label</p>
        <p className="light-gray no-margin-top">Court, clair, 1 à 2 mots max. Majuscule initiale sans ponctuation.<br />
        Ex : "En cours", "Urgent", "Facturé"</p>

        <p className="no-margin-bottom">Icône</p>
        <p className="light-gray no-margin-top">Uniquement décorative ou symbolique. Elle ne remplace jamais le texte.</p>

        → Ils n’incluent pas les “Behaviors” qui correspondent à comment le composant réagit.</p>

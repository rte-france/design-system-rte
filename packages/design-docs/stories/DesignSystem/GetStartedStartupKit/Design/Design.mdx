                        <li>Grille fixe ou fluide selon le produit</li>
                        <li>Marges et colonnes conformes aux tokens</li>
                        <li>Layout guides pour maintenir la structure</li>
                    </ul>
                </div>

                <div>
                    Avant de créer un écran :
                    <ol>
                        <li>Cherchez d’abord le composant dans les Assets</li>
                        <li>Vérifiez ses variantes existantes</li>
                        <li>Consultez les spécifications pour connaître ses comportements et usages</li>
                    </ol>
                </div>
            </DocsSection>

            <DocsSection title="Adapter ensuite avec les Composants Métiers">

            </DocsSection>
        </Accordion>

        <Accordion title="Respecter les règles d’usage et les bons comportements">
            <DocsSection>
                Chaque composant du Design System est documenté avec :

                <ul>
                    <li>Ses comportements (états, interactions, règles d’accessibilité)</li>
                    <li>Ses options (variants, features, propriétés)</li>
                    <li>Ses do/don’t</li>
                    <li>Ses cas d’usage recommandés</li>
                </ul>

                Avant d’utiliser un composant, <strong>ouvrez sa documentation DS</strong> pour confirmer que vous l’utilisez correctement
            </DocsSection>

                        <li>Ne dupliquez pas un composant ou ne modifiez pas ses tokens visuels — cela crée des incohérences dans tout l’écosystème.</li>
                    </ul>

        <Accordion title="Bonnes pratiques pour bien démarrer">
            <DocsSection>
                <ul>
                  <li>Toujours commencer par consulter la documentation du composant</li>
                  <li>Ne pas modifier les tokens (couleurs, radius, spacing, typographies…)</li>
                  <li>Favoriser les layouts compatibles avec les grilles du DS</li>
                  <li>Utiliser les variants, ne pas “casser” les composants Figma</li>
                  <li>Vérifier systématiquement l’accessibilité (contraste, focus, clavier)</li>
                  <li>En cas de doute → contactez l’équipe Design System</li>
                </ul>

<h1 id="getting-started" style={{fontFamily:"Nunito"}}> Démarrage avec Angular </h1>

<h2 id="introduction" style={{border:"none", fontFamily:"Nunito"}}> Introduction </h2>

<p> Le **Design System RTE** est une bibliothèque de composants **Angular** qui permet d'unifier le comportement et l'apparence des interfaces utilisateur à travers les équipes de **RTE France**. </p>

<p> Il est organisé en monorepo avec trois packages principaux. Le package <code>@design-system-rte/core</code> contient les design tokens et interfaces partagées, tandis que <code>@design-system-rte/angular</code> fournit les composants **Angular** spécifiques. Cette architecture permet une maintenance centralisée tout en offrant des implémentations optimisées pour chaque framework. </p>

<h2 id="installation" style={{border:"none", fontFamily:"Nunito"}}> Installation </h2>

<p> Pour installer le design system dans votre projet **Angular**, exécutez la commande suivante : </p>

```bash
npm install @design-system-rte/angular
```

<p> Le package <code>@design-system-rte/core</code> sera automatiquement installé comme dépendance pour accéder directement aux design tokens et interfaces **TypeScript**. </p>

<h3 id="prerequisites" style={{border:"none", fontFamily:"Nunito"}}> Prérequis et Peer Dependencies </h3>

<p> Ce guide suppose que vous avez déjà une application **Angular** existante. La version d'**Angular** doit être **18** ou supérieure.<br/>Le package utilise aussi les peer dependencies suivantes (qui devraient déjà être présentes dans votre projet) :</p>

<ul>
  <li> <code>@angular/common : ^18.0.0 </code></li>
  <li> <code>@angular/core : ^18.0.0 </code></li>
  <li> <code>sass : ^1.85.0 </code></li>
</ul>


<h2 id="setup-configuration" style={{border:"none", fontFamily:"Nunito"}}> Configuration initiale </h2>

<h3 id="add-theme-mixin" style={{fontFamily:"Nunito"}}> Ajouter le mixin Branding/Theme </h3>

<p> Incluez le mixin **branding/theme** dans votre fichier `.scss` principal (généralement `styles.scss` à la racine de votre projet). Cela créera une portée de style pour les composants. </p>

```scss
@use "@design-system-rte/core/design-tokens/tokens/mixins" as *;

@include theme-selector('bleu_iceberg', 'light');
```

<p> Les thèmes disponibles sont : </p>

<ul>
  <li> <code>bleu_iceberg</code> (thème par défaut) </li>
  <li> <code>vert_foret</code> </li>
  <li> <code>violet</code> </li>
</ul>

<p> Chaque thème supporte les modes <code>light</code> et <code>dark</code>. </p>

<h3 id="add-html-selector" style={{fontFamily:"Nunito"}}> Ajouter les sélecteurs HTML </h3>

<p> Ajoutez les attributs <code>data-theme</code> et <code>data-mode</code> sur l'élément <code>&lt;body&gt;</code> de votre application pour définir le contexte d'application des styles des composants. </p>

```html
<body data-theme="bleu_iceberg" data-mode="light">
  <app-root></app-root>
</body>
```

<p> Ces attributs peuvent être modifiés dynamiquement pour changer de thème ou de mode (clair/sombre) à l'exécution. </p>

<h3 id="icons-setup" style={{fontFamily:"Nunito"}}> Configuration des icônes </h3>

<p> Pour utiliser le composant Icon (<code>rte-icon</code>), vous devez copier les fichiers `.svg` depuis le package core vers votre application. </p>

<p> Copiez les fichiers depuis <code>node_modules/@design-system-rte/core/assets/icons</code> vers le répertoire <code>/assets/icons</code> de votre application. </p>

<h2 id="quick-example" style={{border:"none", fontFamily:"Nunito"}}> Exemple d'utilisation </h2>

```typescript
import { Component } from '@angular/core';
import { ButtonComponent } from '@design-system-rte/angular';

@Component({
  selector: 'app-example',
  imports: [ButtonComponent],
  template: `
    <button 
      rteButton 
      rteButtonVariant="primary"
      rteButtonSize="m"
      (click)="handleClick()"
    >
      Click me
    </button>
  `
})
export class ExampleComponent {
  handleClick() {
    console.log('Button clicked!');
  }
}
```

<p> Vous pouvez lire le reste de la documentation du composant **Button** (<code>rteButton</code>) sur le <a href="https://opensource.rte-france.com/design-system-rte/?path=/docs/angular_button--docs">Storybook du composant</a>.</p>

<h2 id="theming-customization" style={{border:"none", fontFamily:"Nunito"}}> Thématisation et personnalisation </h2>

<h3 id="themes-available" style={{fontFamily:"Nunito"}}> Thèmes disponibles </h3>

<p> Le design system RTE propose trois thèmes de marque : </p>

<ul>
  <li> <code>bleu_iceberg</code> : Thème par défaut avec une palette de bleus </li>
  <li> <code>vert_foret</code> : Thème avec une palette de verts </li>
  <li> <code>violet</code> : Thème avec une palette de violets </li>
</ul>

<p> Chaque thème supporte les modes clair (<code>light</code>) et sombre (<code>dark</code>). </p>

<h3 id="switch-theme" style={{fontFamily:"Nunito"}}> Changer de thème </h3>

<p> Pour changer de thème, modifiez les attributs <code>data-theme</code> et <code>data-mode</code> sur l'élément qui heberge le thème.</p>

```html
<!-- Thème bleu_iceberg en mode clair -->
<body data-theme="bleu_iceberg" data-mode="light">

<!-- Thème vert_foret en mode sombre -->
<body data-theme="vert_foret" data-mode="dark">
```

<p> Vous pouvez également changer ces attributs dynamiquement pour changer le thème ou mode pendant l'exécution de l'application. </p>
